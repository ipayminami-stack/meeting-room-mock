# 若手エンジニア向け：Web公開（デプロイ）ガイド

作成したモックアプリケーションを、社外の人や自分のスマホからアクセスできるように「Web上に公開（デプロイ）」してみましょう。
Next.js と相性の良い **Vercel** を使う方法が最も簡単で推奨されています。

---

## 1. 仕組みの理解

*   **ローカル環境**: あなたのPC内でのみ動いています (`localhost:3000`)。他からは見えません。
*   **GitHub**: ソースコードをインターネット上に保存する場所です。
*   **Vercel**: GitHubにあるコードを読み込んで、世界中からアクセスできるサーバーに自動で配置（デプロイ）してくれるサービスです。

**流れ:** `PC` → (Push) → `GitHub` → (Auto Deploy) → `Vercel` (Web公開)

---

## 2. 公開手順

### 手順1: GitHubにリポジトリを作成する
1. [GitHub](https://github.com/) にログインし、新しいリポジトリを作成します（例: `meeting-room-mock`）。
2. VS Codeでターミナルを開き、以下のコマンドでコードをアップロードします。
   ```bash
   # まだ初期化していない場合
   git init
   git add .
   git commit -m "Initial commit"
   
   # 作成したリポジトリに関連付け (URLは自分のものに変えてください)
   git remote add origin https://github.com/<your-username>/meeting-room-mock.git
   git branch -M main
   git push -u origin main
   ```

### 手順2: Vercelと連携する
1. [Vercel](https://vercel.com/) にアクセスし、「Sign Up」から **GitHubアカウントでログイン** します。
2. ダッシュボードの「**Add New ...**」→「**Project**」を選択します。
3. 手順1で作成した `meeting-room-mock` リポジトリが表示されるので、「**Import**」ボタンを押します。
4. 設定画面が開きますが、Next.jsの場合は基本的に **設定不要** です。そのまま「**Deploy**」を押します。

### 手順3: 完成！
1. 数分待つと「Congratulations!」と表示されます。
2. 表示されたURL（例: `https://meeting-room-mock.vercel.app`）をクリックすると、Web上でアプリが動いているのが確認できます。
3. このURLを共有すれば、誰でもモックを確認できます。

---

## 3. 注意点（重要）

このモックアプリは**データベースを使っていません**。
データはメモリ上（ブラウザを開いている間の一時的な記憶）にあるため、以下の挙動になります：

*   **ページをリロードすると、予約データは初期状態に戻ります**。
*   Aさんが予約申請しても、Bさんの画面にはリアルタイムでは反映されません（BさんはBさんの世界で動いています）。

**デモを行う際のアドバイス:**
「あくまで画面イメージと操作感を確認するためのモック（張りぼて）ですので、入力したデータは保存されません」と一言添えて共有するとスムーズです。
