# Walkthrough - 会議室予約システムモック

本プロジェクトでは、「会議室予約システム」のモックを作成し、若手エンジニア向けの環境構築ガイドを整備しました。

## 成果物

### 1. 若手エンジニア向けガイド
- **場所**: `docs/junior_guide.md`
- **内容**: リポジトリのクローンから実行までの手順、ディレクトリ構成の意図、推奨ツールなどを解説。

### 2. モックアプリケーション
- **実装機能**:
  - **ログイン/ロール選択**: 申請者、承認者、管理者、監査役の視点を切り替え可能。
  - **ダッシュボード**:
    - **申請者**: カレンダー形式での空き状況確認、新規予約申請。
    - **承認者**: 未処理申請のリスト表示、承認/却下アクション。
  - **UI**: コンポーネント指向（`src/components/ui`）により、一貫したデザインを実現。

### 3. Web公開ガイド
- **場所**: `docs/deployment_guide.md`
- **内容**: Vercelを使用したデプロイ手順の解説。


## 動作確認手順

1. **開発サーバーの起動**
   ```bash
   npm run dev
   ```
   [http://localhost:3000](http://localhost:3000) にアクセスします。

2. **シナリオ実行例**
   1. トップページで「**叶会 太郎 (申請者)**」を選択します。
   2. カレンダーの「**＋ (空き)**」をクリックします。
   3. モーダルで「利用目的」「人数」を入力し、「**申請する**」をクリックします。
   4. 「**自分の予約**」リストに「**申請中**」として追加されたことを確認します。
   5. 右上のログアウトボタンでトップに戻ります。
   6. 「**吉田 課長 (承認者)**」を選択します。
   7. 未処理リストに先ほどの申請が表示されていることを確認します。
   8. 「**承認**」ボタンをクリックします。
   9. 「**処理履歴**」タブで、ステータスが「**承認済**」になったことを確認します。
